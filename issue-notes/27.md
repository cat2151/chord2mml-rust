# issue CIがエラー #27
[issues #27](https://github.com/cat2151/chord2mml-rust/issues/27)

## 問題の原因

deploy-pages.yml ワークフローが WASM ビルドを試行するが、tree-sitter クレートが wasm32-unknown-unknown ターゲットにコンパイルできないため失敗していた。

## 根本原因

- tree-sitter クレート (v0.20-v0.26+) は wasm32-unknown-unknown ターゲットに対応していない
- C標準ライブラリヘッダー (stdio.h, stdlib.h等) が wasm32-unknown-unknown 環境では利用できない
- これは tree-sitter のアーキテクチャ上の制限であり、バージョンアップや設定変更では解決できない

## 実装した解決策

1. `.github/workflows/deploy-pages.yml` ワークフローを削除
2. 不要になった WASM ビルド用の設定・依存関係を整理
3. 全テストが通ることを確認
4. ネイティブビルドが正常に動作することを確認

## 影響

- GitHub Pages による Web デプロイは非対応となった
- CLI とコアライブラリは問題なく動作
- chord2mml-wasm と chord2mml-web ディレクトリは残るがデプロイされない
